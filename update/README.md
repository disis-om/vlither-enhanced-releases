# Signed GitHub Update Metadata

Vlither Enhanced reads its active update pointer from:

```text
https://raw.githubusercontent.com/disis-om/vlither-enhanced-releases/main/update/latest.json
https://raw.githubusercontent.com/disis-om/vlither-enhanced-releases/main/update/latest.json.sig
```

`latest.json` contains the official release identity, version, immutable GitHub
APK URL, exact byte size, SHA-256 digest, changelog summary and OM Rajput
creator credit. `latest.json.sig` is a detached ECDSA signature over the exact
manifest bytes.

The Android client verifies that signature with its bundled public key before
trusting any update information. It then independently verifies the downloaded
APK size, SHA-256, package identity, version code and Android signing
certificate before opening the system installer.

## Active version

- Version: `1.1.0`
- Version code: `10100`
- Source: GitHub Releases
- Mandatory: no

## Immutability

- Versioned snapshots are preserved under `update/history/`.
- Historical manifests and signatures must never be replaced.
- A correction is published with a newer version code.
- Signed files are marked `-text` in `.gitattributes` because automatic line
  ending conversion would invalidate their signatures.

## Publishing

Signed metadata is generated by the local publisher under
`vlither enhanced backend/github-update-publisher`. The EC private key is kept
outside this repository and outside the shared workspace. These files must
never be edited manually.
