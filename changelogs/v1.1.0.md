# Vlither Enhanced 1.1.0

## The Foundation Release

**Released:** 18 July 2026  
**Created and maintained by:** **OM Rajput**  
**Android version code:** `10100`

Vlither Enhanced 1.1.0 is not a routine revision. It is the first public,
production-signed statement of what this client is meant to become: a native
Android arena experience with its own interface language, mobile-first control
system, measured graphics profile and an update architecture built for the
long road ahead.

This release retires the fragmented experimental phase and establishes one
clean public baseline. From this point forward, every improvement has a known
version, a verifiable binary and a permanent place in the release history.

## A complete interface reconstruction

- Introduced the **Vlither Enhanced** graphite-and-emerald visual identity.
- Rebuilt the Home screen around a focused command panel with product branding,
  player statistics, nickname and server fields, and a dominant Play action.
- Unified Settings, Controls and Update Center under the same spacing, surface,
  border, typography and touch-target system.
- Added a restrained grid-and-orb arena backdrop that gives non-game screens
  depth without competing with the controls.
- Added clear version identity in the interface so testers can immediately
  confirm the build they are running.
- Preserved the original renderer-driven Skin Editor preview, palette and
  accessory workflow while bringing its controls into the shared visual system.

The result is a client that no longer feels like a desktop utility forced onto
a phone. Its navigation, hierarchy and visual rhythm now belong to Android.

## Mobile controls built as a system

- Added selectable **left-handed** and **right-handed** layouts.
- Added **floating** and **fixed** joystick behavior.
- Added **touch-zone** and **fixed-button** boost behavior.
- Added draggable joystick, boost and zoom positions with a dedicated layout
  editor.
- Added zoom orientation selection, control sizing and interface opacity.
- Added reset, cancel and save flows for safe layout experimentation.
- Moved the control editor action outside the scrolling region and made it
  activate on touch-down, preventing small finger movement from cancelling the
  action as a scroll gesture.

These controls are a foundation rather than a dead end: the settings model and
dedicated Controls surface are ready to accommodate future mobile actions.

## The renderer, made more realistic for phones

- Reduced the arena background workload to a **2048 × 1774** asset.
- Reduced the sprite atlas footprint to **3136 × 4032**, lowering the largest
  texture dimension from the former desktop-class 8K target.
- Retained mipmapped background rendering for stable scaling across zoom levels.
- Kept the tightly packed sprite atlas on a non-mipmapped sampling path to
  prevent distant sprites from bleeding colors from neighboring atlas cells.
- Added guarded texture decoding and fallback behavior so a failed resource is
  reported instead of being used as an invalid upload source.
- Preserved the native Vulkan pipeline and arm64 build while substantially
  reducing packaged APK size compared with earlier internal builds.

This is a compatibility-first optimization pass: lower memory pressure without
turning the arena into a visibly degraded mobile copy.

## GitHub Update Center

- Added **GitHub Updates** as the active public update source.
- Added a non-blocking update-check flow with visible checking, availability,
  download, verification and installation states.
- Added strict release identity checks for package name, version code, download
  host and expected asset path.
- Added APK byte-size and SHA-256 validation before installation.
- Added production-certificate continuity checks so an APK signed by an
  unrelated key is rejected.
- Reserved **Official Publisher Updates** as a disabled future source without
  allowing it to issue background requests.

The updater is designed around explicit user approval. It does not silently
replace the application in the background.

## Backup Vault

- Added user-owned backups through Android's document provider.
- Captures nickname, skin, gameplay, graphics and control preferences.
- Added Create Backup, Check for Backups and Restore Latest Backup actions.
- Added integrity-aware `.vebackup` handling with bounded settings payloads.
- Future update installation can request a backup before download begins.
- The selected shared backup location survives ordinary app uninstall/reinstall
  cycles when the user keeps that folder.

## Reliability and platform work

- Standardized package identity as `com.vlither.enhanced`.
- Set the public baseline to Android 8.0 / API 26 and newer.
- Shipped a focused `arm64-v8a` native binary set.
- Preserved Vulkan resource ownership, SDL3 Android lifecycle integration and
  the existing Slither protocol behavior.
- Kept update and backup work off the render/UI thread so the title screen and
  Play control remain responsive during network operations.
- Completed native compilation, Gradle packaging, APK signature verification
  and physical OnePlus launch/crash-buffer checks for the release candidate.

## Before installing

- This build is for 64-bit ARM Android devices with Vulkan support.
- Install only the APK attached to the official `v1.1.0` GitHub Release.
- Android will ask for permission to install apps from the browser or file
  manager used to open the APK.
- Vendor GPU drivers differ. Device-specific rendering reports should include
  the phone model, Android version and GPU name.

## What comes next

**Vlither Enhanced 1.2.0**: new work will build on this verified package,
